Curso Git

Sistema de control de versiones

Areas de Git

working Directory

Es la carpeta en la que iniciamos el repositorio, donde realizamos los cambio en los archivos y demas elementos y muy rara vez estaremos utilizando la consola.

Staging Area (INDEX)

Es el area de control de cambio, un archivo que identifica y verifica los cambios realizados en el directorio (carpeta) de trabajo. Todo archivo al que querramos hacerle seguimiento debe ser agregado previamente al INDEX para que se realice su snapshot.

Repository (local / remote)

Es el area donde se almacena los cambios confirmados en formato BLOB (binary Object File. Esto nos permite reestablecer tanto archivo, directorios, o hasta el mismo proyecto utilizando como referencia dichos archivos e indicando a git en que instacia del mismo.
